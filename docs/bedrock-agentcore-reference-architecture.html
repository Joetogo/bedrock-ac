One conversational agent that reads meeting-room telemetry, Microsoft Teams call quality, and Cisco ThousandEyes network tests — and correlates them on demand. It now runs entirely in AWS: the agent is hosted on Amazon Bedrock AgentCore Runtime, and a browser web console is live on CloudFront — no laptop in the loop. The one-paragraph version

The system is an MCP server hosted by an AgentCore Gateway. Four AWS Lambda functions are exposed to a Claude model as read-only tools; the Gateway injects each upstream's credentials server-side, so the model never sees the Neat API key, the Graph client secret, or the ThousandEyes token. Neat, ThousandEyes and Microsoft Teams call records all answer real queries end-to-end today. The driver loop is no longer on a laptop — it runs as a hosted AgentCore Runtime agent (Strands + Memory), always-on and reachable through a browser web console (Next.js SPA on CloudFront, Cognito login). The agent answers in clean markdown and proactively renders interactive charts. Everything below is the map, the risks, the optimisations, and the toolsets worth adding next — each rated for how much effort it takes.

Logical view — one request, end to end

The same picture as a lifecycle, so the trust boundaries are explicit.

  • Ask. The operator sends plain English. The driver mints a Cognito client-credentials token and opens a streamable-HTTP MCP session to the Gateway.
  • Discover. The model receives the Gateway's tools/list — nine tool specs with JSON schemas — as its tool config.
  • Reason + call. Bedrock decides which tools to call. Each call is a JWT-authorised MCP request to the Gateway, which invokes the target Lambda under its own IAM role.
  • Fetch, secretly. The Lambda reads only its secret from Secrets Manager, calls the upstream, and returns compact JSON. The secret never leaves the function.
  • Answer. Tool output re-enters the model as untrusted context; the loop repeats until the model stops calling tools, then returns prose — with the standing instruction that correlation is observational, not causal.

Two independent auth hops, by design. Caller→Gateway is one Cognito client protecting the MCP endpoint. Gateway→Lambda→upstream is per-service secret injection. Collapsing these — e.g. handing upstream keys to the caller — is the anti-pattern this architecture exists to avoid.

Credential injection · src/_shared/clients.py

# The model never sees any of these. Each Lambda reads only its own secret,
# calls the upstream with a stdlib request, and returns plain JSON.
def te_get(path, params=None):
    sec = get_secret(os.environ["THOUSANDEYES_SECRET_ARN"])   # Secrets Manager
    status, payload = _http("GET", f"{TE_BASE}{path}", {
        "Authorization": f"Bearer {sec['bearer_token']}",   # injected server-side
        "Accept": "application/json",
    })
    return payload   # compact — it re-enters the model as untrusted tool output

Hosted entrypoint · agent/agent.py

# The whole local loop collapses into a hosted entrypoint.
app = BedrockAgentCoreApp()

@app.entrypoint
def invoke(payload):                    # {"prompt", "sessionId"}
    try:
        return answer(payload)             # recall memory → Strands tool-loop → save turn
    except Exception as e:
        return {"error": str(e), "sessionId": payload.get("sessionId")}

# Memory degrades to stateless on failure — a memory hiccup never blocks an answer.

Read-only is preserved by construction through the uplift. The hosted agent adds no tools; it can only reach the Gateway's nine GET-shaped tools. There is no create/update/delete path anywhere in scope — a property worth stating plainly to a risk reviewer.

04

Things to look out for

Ordered by what a bank's risk and platform teams will raise first. Severity uses the coverage-map language: red = block, amber = fix before scale, green = hygiene.

critical

Data residency — the whole stack is pinned to us-east-1

Region was chosen to sidestep AgentCore/Bedrock availability questions, not for data policy. For an ANZ banking audience, meeting-room, call-quality and network telemetry landing in N. Virginia is a sovereignty conversation on its own. Neat's own API is in Norway, adding a second cross-border hop.

ActionConfirm AgentCore Gateway + Runtime + your Claude models are available in ap-southeast-2 (Sydney), then re-home. Treat the Neat-in-Norway egress as a documented exception.

critical

Graph CallRecords.Read.All is a tenant-wide, app-only grant

callRecords does not support delegated auth — the app can read every Teams call in the tenant. That secret is effectively an organisation-wide surveillance key. In a bank this needs an explicit owner, justification, and blast-radius sign-off.

ActionRotate on a schedule, store nowhere but Secrets Manager, and document the grant in the risk register. Consider a dedicated app registration used by nothing else.

fix before scale

Layer architecture mismatch in the SAM template

Globals.Function.Architectures is [x86_64] (switched from arm64 to dodge a Windows Docker build deadlock), but SharedLayer.CompatibleArchitectures still reads [arm64]. Pure-Python means it works today, but the two disagree — and this is the same file whose layer double-nesting once made three functions silently fail to import.

ActionMake them agree. Best is to move builds to CodeBuild/Linux and standardise on arm64 (Graviton) for both — cheaper and consistent (see §05).

fix before scale

Single-operator inbound auth won't survive real users

The planned Runtime is reached by SigV4 as one operator — fine for a POC. The moment more than one human, or a UI, needs access, you need real identity, not shared IAM.

ActionAdopt AgentCore Identity for inbound OAuth (per-user tokens) before opening it up. It also centralises the outbound token handling you currently hand-roll (see §06).

fix before scale

Correlation join is name-substring matching

The POC ties a room to its calls by matching name substrings. Two rooms called "Level 3 Boardroom" in different buildings will cross-contaminate. The team has already decided the fix.

ActionShip the planned config/locations.json explicit map (Neat space id ↔ Teams resource account ↔ ThousandEyes site). Missing ids degrade to null, not to a wrong match.

fix before scale

Tool output is untrusted model context — a prompt-injection surface

The code comments already flag it: upstream JSON re-enters the model. A malicious room name or test label could carry instructions. Low risk on first-party data, non-zero at a bank.

ActionAdd Bedrock Guardrails for PII redaction + injection heuristics, and keep tool payloads compact (already done). See §06.

hygiene

Secret caching outlives rotation

get_secret uses an lru_cache that persists for the life of a warm Lambda container. After a rotation, warm containers keep serving the old value until they recycle.

ActionAdd a short TTL to the cache (e.g. 5–15 min) so rotation propagates without a cold start. Trivial change, avoids a confusing outage.

hygiene

Template ships REPLACE_ME secret placeholders + model EOL churn

Secrets deploy with literal REPLACE_ME values (populated out-of-band). And Bedrock model versions reach EOL — hardcoding an id will break. Both are already handled operationally, but both are easy to trip over on a fresh deploy.

ActionKeep the auto-resolve-newest-Sonnet logic (already in place); add a post-deploy check that fails loudly if any secret still reads REPLACE_ME.

05

Optimisations worth doing

Cheap wins first. None of these change the read-only contract.

  • Turn on AgentCore Observability. Near-zero effort, largest operational payoff. Strands emits OpenTelemetry natively; point it at CloudWatch GenAI and you get per-session traces, tool latency, and token spend — the difference between "the agent is slow" and "the te_path_visualization call is slow".
  • Prompt-cache the static prefix. The system prompt and nine tool schemas are identical every turn. Bedrock prompt caching on that prefix cuts input-token cost materially on a chatty tool-use loop.
  • Enable Gateway semantic tool search. At nine tools the full list is fine. As you add Webex, ServiceNow, Meraki, etc., semantic search lets the model pull only the relevant tools into the prompt — smaller prompts, lower latency, lower cost.
  • Move Lambdas to arm64 (Graviton). ~20% cheaper for the same pure-Python workload, and it resolves the §04 architecture mismatch. Needs a Linux/CodeBuild build step so you're not fighting Windows Docker emulation.
  • Add long-term Memory strategies later. Short-term event store covers "that first room". A semantic/summary strategy would let the agent remember standing facts across sessions ("Level 3 Boardroom always spikes at 2pm") — opt in when the use case appears; it costs per stored memory.
  • Give correlate a TTL cache + parallel fetch. It fans out to multiple upstreams per call; caching room lists and issuing concurrent requests shortens the slowest path in the system.
